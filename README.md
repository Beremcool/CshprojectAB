# TaskHub

Это мой финальный проект по C#. Я сделал обычное консольное приложение для работы с задачами. Через меню можно добавить задачу, посмотреть список, изменить данные, удалить задачу, найти нужную и сохранить всё в файл.

Проект называется **TaskHub**. Данные сохраняются в `tasks.json`, поэтому после перезапуска программы задачи можно загрузить обратно.

## Как запустить

Нужен установленный .NET SDK 8.0 или новее.

```bash
cd TaskHub
dotnet run
```

После запуска появится меню. Нужно просто вводить номер пункта и следовать подсказкам в консоли.

## Что умеет программа

В приложении можно:

- создать задачу с названием, описанием, приоритетом, дедлайном и статусом;
- посмотреть все задачи;
- отдельно посмотреть выполненные, невыполненные и задачи с высоким приоритетом;
- отредактировать задачу по ID;
- удалить задачу;
- отметить задачу как выполненную;
- искать задачи по названию, статусу или приоритету;
- посмотреть статистику;
- сохранить задачи в файл;
- загрузить задачи из файла.

Еще есть фоновая проверка дедлайнов. Она работает отдельно от основного меню и раз в 10 секунд проверяет, появились ли просроченные задачи. Если появились, программа выводит уведомление в консоль.

## Формат дедлайна

Дату можно вводить в нескольких форматах, чтобы было удобнее:

```text
yyyy-MM-dd HH:mm
yyyy-MM-dd
dd.MM.yyyy HH:mm
dd.MM.yyyy
```

Например:

```text
2026-05-30 18:00
2026-05-30
30.05.2026 18:00
30.05.2026
```

## Как примерно устроен проект

Я постарался не писать всю программу в одном `Program.cs`, поэтому код разделен по папкам:

```text
Models/        модели и enum'ы
Services/      основная логика задач, статистика и проверка дедлайнов
Repositories/  сохранение и загрузка из JSON
Utilities/     ввод и вывод в консоль
Delegates/     делегат для фильтрации задач
```

Главные классы:

- `TaskItem` — сама задача;
- `TaskManager` — хранит список задач и выполняет основные операции;
- `TaskHubApp` — консольное меню;
- `JsonFileRepository<T>` — работа с файлом;
- `DeadlineMonitor` — фоновая проверка просроченных задач;
- `StatisticsService` — подсчет статистики.

## Что использовано из тем C#

В проекте есть основные темы, которые требовались:

- ООП и классы — логика разделена на отдельные классы;
- `List<TaskItem>` — список задач;
- `Dictionary<Priority, int>` — статистика по приоритетам;
- generic — `IRepository<T>`, `JsonFileRepository<T>`, `ReadEnum<TEnum>()`;
- делегат `TaskFilter` — используется для фильтрации задач;
- `try-catch` — обработка ошибок при работе программы и с файлами;
- `IDisposable` — репозиторий и фоновый монитор освобождают ресурсы;
- `static` методы и классы — например, `ConsoleHelper` и `StatisticsService`;
- `async/await` — сохранение, загрузка и статистика сделаны асинхронно;
- многопоточность — проверка дедлайнов работает через `Task`;
- `lock` — используется, чтобы список задач не менялся одновременно из разных потоков.

## Файл с задачами

При сохранении создается файл:

```text
tasks.json
```

Он лежит рядом с запускаемым приложением. Если файла еще нет, при загрузке программа просто вернет пустой список задач.
